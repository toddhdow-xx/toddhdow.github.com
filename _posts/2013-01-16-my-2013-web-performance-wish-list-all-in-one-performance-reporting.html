I'm a huge fan of Compuware's Gomez product. It's like Google's Pagespeed (specifically the PageSpeed Critical Path Explorer) on steroids. I've been using Gomez for a few years now and it is one of the most helpful tools that I've found for reliably measuring site performance over time.

Gomez provides the ability to do real time monitoring of websites, and it provides detailed performance data over a rolling 30 day period. This data can be sliced and diced in many different ways. Here are just three examples of some useful reports:

These reports allow me to compare my site's performance with that of my competitors. And, it also allows me to quickly and easily identify the sources of show loading websites.

This has been a tremendous help in troubleshooting performance issues as it has allowed us to quickly identify which object(s) on a page are slow loading. From there, we can focus our attention on those slow loading pieces to see what is going on. In some cases, the culprit is an overly taxed server - adding additional resources usually helps. In other cases, there is a bug in a recent code release - a rollback and a bit of extra time tweaking the code fixes these issues. And in other cases, network issues are at play. Regardless of the root cause, this tool is very helpful in pointing us in the right direction.

The one spot that remains a weakness with Gomez, Google PageSpeed or any of the various web-based page speed plug-ins is this: what is going on inside of those really long "time bars"? Each object takes a discrete amount of time to load, and these tools show us how long that object took to get to our browser from the time we requested it until the time it finally arrived. But what really happened in there? There are server calls, CPU time, etc. involved in each call. If we find a performance issue in Gomez, we then have to dive into server performance logs and other tools to find out what's going on.

I'd love to see the marrying of these two kinds of data. And it is coming.

Compuware has been working hard on another product called Dynatrace, which is a back-end server reporting tool. Compuware is working on the integration between Gomez and DynaTrace so that you can see all of the performance data in one place. I suspect that others are working on similar tools.
